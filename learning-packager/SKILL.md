# Learning Packager

## Purpose

把任何**原始学习资料**（PDF 书籍章节、长文章、教程笔记）一次性打包成**内容三件套**，省去人工拆解：

1. **深度解读 md** — 结构化整理 + 类比 + 表格 + 决策树
2. **口播文案稿 md** — 9~11 分钟口语化逐字稿，可直接录音
3. **配图提示词 md** — 8~15 张信息密集型图片的英文 prompt + 生成地址

适用场景：技术博主、公众号运营、知识付费内容生产者、AI 讲师。

## Trigger

用户说以下任意一句时立即触发：

- "把这个学习资料整理成三件套"
- "帮我做内容三件套"
- "learning-packager"
- "想做成口播 + 配图"
- "把这个 PDF 整理一下发公众号"
- 提供一个 PDF/文件 + "整理 + 口播 + 配图"

## Workflow

### 1. 提取原文（自动）

**PDF 类**：

```bash
which pdftotext && pdftotext -layout "<file.pdf>" /tmp/source.txt
```

> 优先用 `pdftotext`（layout 模式保留排版）。Read 工具对 PDF 返回 binary，无法解析。

**网页 / md / txt**：直接 Read 或 Grep 即可。

**长文章**：先 Read 全文，再 Grep 切章节。

提取后用 Grep 定位章节边界：

```bash
grep -n "^第[0-9]" /tmp/source.txt | head -20
```

### 2. 询问用户（一次性，≤ 3 个问题）

用 AskUserQuestion，**不要超过 4 个问题**：

| 问题 | 必填 | 选项 |
|------|------|------|
| 处理范围 | ✅ | 全篇 / 指定章节（用户输入章节号或关键词） |
| 内容深度 | ✅ | 通俗版（重类比） / 标准版（重技术）/ 进阶版（重代码） |
| 输出语种 | 默认中文 | 中文 / 英文 / 双语 |

> 用户没指定就**默认**："指定章节（最近一章）" + "标准版" + "中文"。

### 3. 生成深度解读 md（输出 1）

按以下骨架生成（章节标题**不含原始资料的书名**，只用主题命名）：

```markdown
# {主题一句话定位}

## 速查表（一页纸地图）
| 模式/概念 | 一句话定义 | 核心比喻 | 典型场景 |

## 0. 全章比方：用生活化隐喻切入
- 一个主比喻覆盖全部核心能力

## {N}.{M} {小节标题}
### 类比：一句话生活化场景
### 痛点：不这么做会怎样
### 架构/原理：核心机制
### 关键工程细节：参数 / 代码片段 / 反例
### 💡 真实数据 / 关键洞察

## 横向对比：决策树 / 矩阵

## 工程踩坑清单（表格）

## 全章知识地图（ASCII 流程图）

## 贯穿主线：一句话哲学总结

## 学习路径建议
```

**命名**：`{主题}-深度解读.md`，存到 `awesome-ai/docs/` 或用户指定目录。

**禁止**：
- ❌ 提到原始资料的书名 / 作者 / 出版社
- ❌ 大段照抄原文（最多引用关键 1~2 句）
- ❌ 每个章节超过 500 字（信息密度优先）

### 4. 生成口播文案稿 md（输出 2）

骨架（**约 2200 字，9~11 分钟**）：

```markdown
# {主题} · 口播文案稿

> 适用场景 / 预计时长 / 建议语速 / 阅读对象

## 【开场钩子 · 30 秒】
（一个反常识问题 + 一句话定位）

## 【{核心比喻开场} · 60 秒】
（用生活场景把 5 大能力铺一遍）

## 【第一道菜 / 第一节 · 90 秒】
## 【第二道菜 / 第二节 · 90 秒】
## 【第三道菜 / 第三节 · 60 秒】
## 【第四道菜 / 第四节 · 80 秒】
## 【第五道菜 / 第五节 · 90 秒】

## 【决策树 · 60 秒】
## 【踩坑提醒 · 60 秒】
## 【收尾 · 30 秒】

## 【播报备注】
- 停顿点 / 重音词 / B-roll 建议 / 章节标记 / 字数 / 时长
```

**风格铁律**：
- 全程口语化（"举个例子"、"说白了"、"对吧"）
- 短句为主，单段不超过 3 行
- 每个技术概念**必须**配一个生活类比
- 不使用 emoji 作为标题

**命名**：`{主题}-口播文案稿.md`。

### 5. 生成配图提示词 md（输出 3）

骨架：

```markdown
# {主题} · 配图提示词

> 用途 / 图片总数 / 生成地址 / 可用尺寸

## 图片 {N}：{主题}（{场景}）
**尺寸**：landscape_16_9 / portrait_4_3 / square_hd
**提示词**：
\`\`\`
A cinematic wide-angle illustration of ...
\`\`\`

## 图片 {N+1}：...

## 配套图片清单（汇总表格）
| # | 主题 | 对应章节 | 尺寸 | 信息密度重点 |

## 使用建议
1. 生成顺序（封面 vs 正文 vs 结尾）
2. 风格统一原则
3. URL 编码提醒
```

**图片数量**：根据内容拆为 **8~15 张**，每张对应一个核心知识点。

**每张图的提示词必须包含**：
1. **场景/主体**（具体到人物/物体/构图）
2. **中文标签**（用括号标注在元素旁，如 `labeled "意图识别"`）
3. **配色**（具体到主色 + 辅色，避免"好看"等虚词）
4. **风格关键词**（editorial / cinematic / flat design / isometric 3D / neon cyberpunk）
5. **构图说明**（16:9 / 1:1 / 4:3 竖版时间线）

**生成地址模板**（用户后续调用）：

```
https://coresg-normal.trae.ai/api/ide/v1/text_to_image?prompt={URL编码}&image_size={尺寸}
```

**命名**：`{主题}-配图提示词.md`。

### 6. 落盘规范

所有输出**统一存到 `awesome-ai/docs/`**（或用户指定目录），命名**只含主题不含书名**：

```
awesome-ai/docs/
├── {主题}-深度解读.md
├── {主题}-口播文案稿.md
└── {主题}-配图提示词.md
```

时间戳不需要（每个资料独立目录即可，资料本身就是唯一标识）。

### 7. 输出汇总

告诉用户：
- 三个文件的绝对路径
- 一句话总结每个文件的核心卖点
- 建议下一步：先生成哪张配图做封面 / 优先发布哪个格式

## Rules

- **绝不**在生成的内容里提到原始资料的书名 / 作者 / 出版社
- **绝不**大段照抄原文——总结 + 类比 + 重组才是价值
- **绝不**让用户回答超过 4 个问题
- **绝不**生成超过 15 张图（信息密度会下降）
- **绝不**把生成物提交到 git——`awesome-ai/docs/` 应在 `.gitignore`
- 用户没说英文就**默认中文**
- 图片 prompt 必须用**英文**写（生成模型对英文 prompt 更稳定）
- 每个生成物单独一份 md，**不要打包成单个文件**（方便单独迭代）

## Anti-patterns

- ❌ 不要把"PDF 转文字"和"内容整理"混为一步——分两步清晰
- ❌ 不要用 `cat` 读 PDF（Read 工具也读不了 binary），必须用 `pdftotext`
- ❌ 不要把类比堆成段——一个概念配**一个**最贴的类比即可
- ❌ 不要给每张图都写"很好看"——必须写具体的色号 / 光线 / 材质
- ❌ 不要生成纯文字的"信息图"——必须有视觉锚点（人物、物体、场景）
- ❌ 不要在口播稿里写"我们来看下一段"这种书面过渡——用"接下来"等口语

## Quick Reference（速查）

| 输入类型 | 提取方式 | 备注 |
|---------|---------|------|
| PDF | `pdftotext -layout` | 优先 `-layout` 保留排版 |
| 长文（md/txt） | Read 全文 | 直接读 |
| 网页 | WebFetch | 注意字符长度上限 |
| 视频字幕 | 先下载 .srt，转 txt | 用 ffmpeg 抽音轨或下载现成字幕 |

| 输出类型 | 字数参考 | 章节数 |
|---------|---------|--------|
| 深度解读 md | 3000~6000 字 | 5~8 节 |
| 口播稿 md | 2000~2500 字 | 8~10 段 |
| 配图提示词 md | 8~15 张图 | 每张 100~200 字英文 prompt |

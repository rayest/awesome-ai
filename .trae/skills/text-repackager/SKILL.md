# Text Repackager（文本二次修改与重组器）

## Purpose

把用户**粘贴 / 复制**过来的文本，按用户指定的命令**二次修改**：

1. **拆解分析**：识别原文结构、核心论点、薄弱环节
2. **重新组织语言系统**：换表达、换节奏、换视角
3. **重新组织文章框架**：重排段落、增删模块、调整逻辑链
4. **联网增强**（按需）：拉取最新数据、多元视角、行业动态
5. **类比优化**：用更贴切的生活化比喻替换生硬表达
6. **系统性思考**：把文本放入更大的系统（行业 / 上下游 / 二阶效应）中审视
7. **结构化思考**：用 MECE 原则 + 因果链 / 矩阵 / 决策树重组论证
8. **批判性 / 思辨性审视**：评估原文观点是否成立、论据是否充分、是否需要补反方视角

**价值定位**：不是简单同义改写，而是**重构 + 升级 + 思辨**——产出物在准确性、精确性、可读性、系统性、**思辨性**上**必须优于原文**。

## Trigger

用户**粘贴文本** + **给出修改命令**时立即触发。常见组合：

| 用户动作 | 触发命令示例 |
|----------|--------------|
| 粘贴一段论文 | "改写得通俗一点" / "加入更多案例" / "审视一下论据够不够硬" |
| 粘贴公众号文章 | "重新组织框架" / "换更正式的语气" / "补上反方观点" |
| 粘贴技术博客 | "加点 2025 年的最新进展" / "补上对比维度" / "从行业视角重看" |
| 粘贴商业文案 | "换更精准的卖点表达" / "加入不同行业的应用案例" |
| 粘贴邮件草稿 | "改成更礼貌的版本" / "精简到 100 字" |
| 粘贴观点性文章 | "批判性审视一下" / "这个观点对吗？" / "用 MECE 重构论点" |

> 用户没指定具体命令时，**默认行为**：拆解 + 重组 + 联网补强 + 类比优化 + 系统性 + 结构化 + 批判性（七件套全开）。

## Workflow

### 1. 解析输入

从用户消息中提取：

| 要素 | 默认行为 | 可覆盖 |
|------|---------|--------|
| 原始文本 | 必填 | 无 |
| 修改命令 | 必填（如缺失，默认全开） | 用户指定 |
| 目标读者 | 与原文相同 | 用户指定（小白 / 专家 / 高管） |
| 目标长度 | 与原文相同 | 用户指定（"精简到 500 字"） |
| 输出语言 | 与原文相同 | 用户指定 |
| 联网开关 | 自动判断（涉及数据 / 趋势时开） | 用户强制 on/off |
| 思辨强度 | 客观呈现（标注存疑 / 不盲目认同 / 不强行否定） | 用户指定（强批判 / 弱批判 / 中立） |

### 2. 拆解分析（X 光扫描）

**对原文做 X 光扫描**（不展示给用户，仅作为内部加工依据）：

| 维度 | 提取内容 |
|------|----------|
| **结构** | 段落数 + 段落功能（引言 / 论点 / 案例 / 结论）+ 段落间逻辑 |
| **核心论点** | 3~5 个关键主张（用一句话概括） |
| **支持论据** | 每个论点的论据来源（数据 / 案例 / 推理 / 类比 / 引用） |
| **隐含假设** | 作者没说但默认成立的前提 |
| **立场倾向** | 原文作者的态度（强烈主张 / 中立陈述 / 反对批评） |
| **薄弱环节** | 数据陈旧 / 论证不足 / 案例单薄 / 逻辑跳跃 / 类比生硬 |
| **可优化点** | 框架重排 / 模块增减 / 视角补全 / 比喻替换 |

### 3. 判断是否联网

**触发联网的条件**（满足任一即开）：

- 原文含具体数据 / 统计 / 时间点（需验证时效性）
- 原文涉及行业趋势 / 技术演进（需最新动态）
- 原文是单方观点（需补多元视角 / 反方论据）
- 原文引用案例过旧（>2 年）
- 修改命令含"最新"、"当前"、"现在"、"补充"、"对比"、"反方"、"反对"

**联网工具**：用 `WebSearch` / `WebFetch` 拉 2~5 条权威源。

### 4. 系统性思考

**把文本放回系统**——不孤立地看待每个观点，而看它在更大系统中的位置：

| 视角 | 思考问题 |
|------|----------|
| **行业背景** | 这个观点在行业里主流吗？是少数派还是共识？ |
| **上下游** | 谁是上游利益方？谁是下游受影响者？ |
| **二阶效应** | 这个观点如果被广泛接受，会引发什么连锁反应？ |
| **历史脉络** | 这个观点是新的还是旧调重弹？前人怎么说过？ |
| **跨学科** | 其他领域（经济学 / 心理学 / 物理学）有没有相似规律？ |

### 5. 结构化思考

**用 MECE 原则 + 工具重组论证**：

| 工具 | 适用场景 | 用法 |
|------|----------|------|
| **MECE 分类** | 列举型内容 | 各模块互不重叠、合起来穷尽 |
| **2x2 矩阵** | 双维度对比 | 紧急/重要 × 难/易等 |
| **因果链** | 现象 → 原因 | A → B → C → 结果 |
| **决策树** | 选择型内容 | if/else 分支可视化 |
| **5W1H** | 流程型 / 说明型 | 何时 / 何地 / 谁 / 什么 / 为什么 / 怎么 |
| **金字塔原理** | 观点型 / 说服型 | 结论先行 → 分论点 → 论据 → 案例 |

### 6. 批判性 / 思辨性审视

**核心理念**：**不盲目认同，也不盲目否定**。每个观点都过 4 道审视：

#### 6.1 评估论据强度

| 论据类型 | 评估标准 | 强度等级 |
|----------|----------|----------|
| **数据 / 统计** | 来源权威？样本量？时效？ | 强 / 中 / 弱 |
| **案例** | 典型性？可复现？时效？ | 强 / 中 / 弱 |
| **类比** | 实质相似？非刻意？ | 强 / 中 / 弱 |
| **推理** | 逻辑自洽？前提成立？ | 强 / 中 / 弱 |
| **专家引用** | 专家权威？引用准确？ | 强 / 中 / 弱 |

#### 6.2 立场判断（4 档）

| 立场 | 触发条件 | 处理方式 |
|------|----------|----------|
| **完全成立** | 强论据 + 多元视角支撑 + 无明显反方 | 直接采用，可强化 |
| **部分成立** | 部分论据不足 / 视角不全 / 有反方 | 采用 + 标注存疑 + 补反方 |
| **存疑** | 隐含假设有问题 / 数据陈旧 / 类比生硬 | 重组论证 + 加新数据 + 标注「需进一步验证」 |
| **不成立** | 论据薄弱 / 逻辑断裂 / 反方有强证据 | 重写为"另一视角"或"质疑式" |

#### 6.3 多角度审视清单

对每个核心观点，问以下 7 个问题：

1. **支持方**：谁会同意？最强论据是什么？
2. **反对方**：谁会反对？最强反方论据是什么？
3. **中立方**：中间立场怎么说？条件成立吗？
4. **历史方**：过去类似观点的结局是什么？
5. **跨国方**：不同国家 / 文化背景怎么看待？
6. **跨学科方**：其他学科怎么解释？
7. **二阶方**：这个观点被普遍接受后会引发什么新问题？

#### 6.4 平衡呈现原则

- **保留核心 + 补反方**：不删除原文合理之处，但主动补反方视角
- **标注存疑**：对存疑论据加「⚠️ 需进一步验证」标记
- **呈现分歧**：当业内有分歧时，并列呈现各方观点
- **不强加立场**：二次修改版本保持中立，不强行站队

### 7. 二次修改原则

| 维度 | 原则 | 反例 |
|------|------|------|
| **准确性** | 数据 / 时间 / 引用必须核实 | 沿用原文错误数据 |
| **精确性** | 每个数字、术语、概念不模糊 | "很多人" / "效果不错" |
| **可读性** | 短句 + 节奏 + 重点突出 | 长段 + 学术腔 + 信息密度失控 |
| **系统性** | 框架完整 + 逻辑自洽 + 模块互不重叠 | 想到哪写到哪 + 同义反复 |
| **结构化** | MECE + 工具辅助（金字塔 / 矩阵 / 决策树） | 信息散乱 + 维度混乱 |
| **思辨性** | 观点必审视 + 论据必核验 + 反方必呈现 | 盲目认同 / 盲目否定 |

### 8. 语言重组策略

| 原文风格 | 重组方向 |
|----------|----------|
| 学术腔 | 拆长句 + 加口语化连接词 + 类比 |
| 公文腔 | 删套话 + 提炼动词 + 量化结果 |
| 技术 jargon 多 | 加白话翻译 + 必要时加括号注释 |
| 故事化但缺结构 | 抽时间线 + 加小标题 + 补数据 |
| 结构清晰但缺血肉 | 加案例 + 加数据 + 加对比 |
| 观点单方 | 加反方视角 + 多角度并列 |

### 9. 框架重组策略

| 原文框架 | 重组方向 |
|----------|----------|
| 总分总（平淡） | 痛点钩子 → 颠覆认知 → 论证 → 升华 |
| 平铺直叙 | 倒叙 / 悬念 / 冲突 → 解决 |
| 单一视角 | 多视角并列 + 横向对比表 |
| 缺首尾 | 加开场钩子 + 收尾升华 |
| 案例散落 | 案例集中化 + 案例 → 洞察 → 应用 三段式 |
| 论点无证据 | 金字塔结构：结论 → 3 个分论点 → 各配 1~2 个论据 |
| 缺反方 | 主观点 → 支持论据 → 反方观点 + 反驳 → 综合判断 |

### 10. 类比优化

| 原文类比 | 优化方向 |
|----------|----------|
| 抽象术语堆砌 | 找一个生活场景（做饭 / 看病 / 装修 / 旅行） |
| 旧类比（>3 年） | 换新类比（外卖 / 短视频 / 直播） |
| 强行类比（不贴切） | 删除或换更贴的 |
| 单一类比贯穿 | 不同模块用不同类比，避免审美疲劳 |

### 11. 输出规范

**不写文件**（除非用户明确说"保存到文件"），**直接在对话中输出**。

**输出结构**（含思辨审视的完整版）：

```markdown
## 二次修改版本

[重组后的完整文本，含 ⚠️ 存疑标注]

---

## 观点审视报告（可选，用户没问可省）

### 核心观点与论据评估

| # | 核心观点 | 论据类型 | 强度 | 立场判断 |
|---|----------|----------|------|----------|
| 1 | [观点 1] | 数据 + 案例 | 强 | 完全成立 |
| 2 | [观点 2] | 推理 | 中 | 部分成立 |
| 3 | [观点 3] | 类比 | 弱 | 存疑 |

### 多角度审视

- **支持方**：[最强支持论据]
- **反对方**：[最强反方论据] / [补的反方视角]
- **二阶效应**：[如果被广泛接受会引发的连锁反应]
- **跨学科视角**：[其他学科怎么看]

### 关键调整

- **保留**：[哪些核心观点 + 合理论据]
- **强化**：[补了哪些数据 / 案例 / 权威引用]
- **存疑标注**：[哪些论据加了 ⚠️ 需进一步验证]
- **反方补充**：[补了哪些反方视角 + 反驳]
- **框架重排**：[原 N 段 → 现 N 段，核心调整]
- **语言系统**：[学术腔 → 口语化] / [公文化 → 故事化]
- **类比替换**：[X → Y]，原因：[更贴切 / 更新 / 更通俗]
- **删除冗余**：[删了哪些套话 / 同义反复]
- **新增模块**：[加了哪些段落 / 表格 / 列表]
```

**用户没要求时**：直接给二次修改版本，不写审视报告。

## Rules

- **绝不**大段照抄原文（最多保留 20% 关键句，其余必须重组）
- **绝不**沿用未经验证的数据 / 时间点（联网核实或标注"待核实"）
- **绝不**为改而改——如果原文已足够好，明确告诉用户"原文已优，建议保留"
- **绝不**丢失原文核心论点（重组不能改主旨）
- **绝不**添加原文没有的虚假案例（联网能补则补，补不上宁可不加）
- **绝不**省略联网环节的判断（涉及数据 / 趋势时必须决策 on/off）
- **绝不**改变原文的目标读者定位（除非用户明确说）
- **绝不**写"以上是我的改写"这类客套话
- **绝不**盲目认同原文观点（每个观点必过 4 道审视）
- **绝不**盲目否定原文观点（反方必须配证据，不能为反而反）
- **绝不**做"复读机"（即便你同意也要补强 / 补反方）
- **绝不**做"杠精"（无证据反驳）
- **绝不**删除原文的合理之处（即便你不同意）
- **绝不**强加立场给二次修改版本（保持中立呈现）
- 用户没说英文就**默认中文**
- 修改命令不明确时，**默认**全开（拆解 + 重组 + 联网 + 类比 + 系统 + 结构 + 思辨）

## Anti-patterns

- ❌ 不要做"同义替换"式改写（用近义词替换原词，没意义）
- ❌ 不要把原文段落顺序倒一下就交差（必须实质性重构）
- ❌ 不要忽略联网——涉及数据 / 趋势时联网是义务
- ❌ 不要使用过时类比（"就像当年的淘宝" → 2026 年已不贴切）
- ❌ 不要为了凑长度加废话（精简优于啰嗦）
- ❌ 不要把"重新组织"理解成"加几个小标题"
- ❌ 不要删除原文的数据 / 案例（要核实或替换，不能凭空消失）
- ❌ 不要"全盘照搬"原文观点（思辨性是义务）
- ❌ 不要"全盘否定"原文（即便你不同意也要保留合理之处）
- ❌ 不要因为观点"主流"就放行（要看论据）
- ❌ 不要因为观点"小众"就否定（要看论据）
- ❌ 不要把"批判"等同于"反驳"（批判 = 评估，不是反对）
- ❌ 不要把"思辨"等同于"骑墙"（思辨 = 客观评估 + 标注 + 补全）

## Quick Reference（速查）

| 输入类型 | 提取方式 | 备注 |
|---------|---------|------|
| 纯文本（粘贴） | 直接读取用户消息 | 必带修改命令 |
| 含 markdown | 解析结构后重组 | 保留或重排格式由用户决定 |
| 链接 / 图片 | 提示用户转文本 | 暂不支持直接处理 |
| 表格 | 保留表格但优化内容 | 必要时加列 / 改顺序 |

| 输出类型 | 字数参考 | 调整范围 |
|---------|---------|----------|
| 同等长度改写 | ±10% | 框架重组 + 语言优化 |
| 精简版 | 原文 30~60% | 砍冗余 + 保核心 |
| 扩展版 | 原文 150~200% | 加案例 + 加数据 + 加对比 |
| 通俗版 | 视原文而定 | 拆 jargon + 加类比 + 加场景 |
| 思辨审视版 | 原文 + 30~50% | 加审视报告 |

## 思辨审视速查表

| 原文特征 | 审视重点 | 处理方式 |
|----------|----------|----------|
| 含数据但无来源 | 数据真实性 | 联网验证 / 标注存疑 |
| 单方主张 | 多元视角 | 联网找反方 + 补并列 |
| 案例 > 2 年 | 时效性 | 联网找新案例 / 标注陈旧 |
| 类比频繁 | 类比质量 | 评估类比强度，弱则删 |
| 因果链长 | 逻辑漏洞 | 拆成 2x2 矩阵或决策树 |
| 隐含假设强 | 假设审视 | 明示假设 + 检验 |
| 立场鲜明 | 客观呈现 | 主观点 + 反方 + 中立三方并列 |

## 依赖

- `WebSearch` / `WebFetch`（按需启用）
- 无本地依赖

## 示例对话

**用户**：
```
[粘贴一段 2023 年的 AI Agent 综述文章]

帮我二次修改，融入 2025 年的最新进展 + 批判性审视一下论据。
```

**本 Skill 自动执行**：
1. 拆解：识别原文 5 段，结构为"概念 → 分类 → 应用 → 趋势"
2. 联网：搜 "AI Agent 2025 latest advances" + "Multi-Agent system 2025" + 反方观点
3. 系统性思考：把 AI Agent 放在整个 LLM 应用生态中看
4. 结构化思考：用 MECE 重构分类
5. 批判性审视：评估每个论点的论据强度，标注存疑
6. 重组：把 2023 年的趋势预测替换为 2025 年实际进展
7. 类比：把过时的类比（"像 PC 互联网"）换成更贴的（"像 App Store 生态"）
8. 输出：完整二次修改版本 + 观点审视报告

**用户**：
```
[粘贴一段技术文档]

让小白也能看懂 + 拆解一下论证逻辑。
```

**本 Skill 自动执行**：
1. 拆解：识别 jargon 列表（"Transformer"、"注意力机制"、"上下文窗口"）+ 提取论证结构
2. 不联网（技术概念无需实时数据）
3. 系统性思考：把 Transformer 放在深度学习演进史中看
4. 结构化思考：用「概念 → 类比 → 例子 → 应用」四段式重组
5. 批判性审视：检查每个技术声明的论据强度

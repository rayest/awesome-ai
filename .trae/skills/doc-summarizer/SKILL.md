# Doc Summarizer（文档摘要与输出）

## 触发条件

当用户的请求符合以下任一情况时，自动调用本 Skill：

- 用户说"总结 / 摘要 / 提炼 / abstract"某个文档（PDF / MD / TXT / 代码文件）
- 用户说"输出为文档 / 写入到文档 / 生成总结文档"
- 用户指定了具体章节（如"第一章"、"第 3 章"、"chapter 2"）
- 用户要求"附带案例 / 示例 / 类比"以方便理解

## 工作流程

### 1. 解析输入

从用户消息中提取以下要素：

| 要素 | 默认行为 | 可覆盖 |
|------|---------|--------|
| 文档路径 | 用户必须提供 | 无 |
| 范围 | 整篇文档 | 用户指定章节、页码、关键词 |
| 输出目录 | `assets/abstract/` | 用户指定 |
| 输出语言 | 与用户输入语言一致 | 用户指定 |
| 是否配案例 | 是（默认开启） | 用户要求关闭 |

如关键信息缺失，**优先做合理推断并继续**，在最终输出中标注所做的假设。

### 2. 文档读取

按扩展名选择读取方式：

| 格式 | 方法 |
|------|------|
| `.pdf` | `pypdf` 库（如未安装：`pip install --user --break-system-packages pypdf`） |
| `.md` / `.txt` | 直接 `Read` 工具 |
| `.docx` | `python-docx` 库 |
| 代码文件 | `Read` 工具 |

读取后先**识别目录 / 大纲**，定位用户指定的章节，提取对应文本块（避免一次性把整本书塞入上下文）。

### 3. 内容提炼原则

- **三层结构**：
  1. 章节主线（一句话 + 演进图）
  2. 核心概念（用对照表 / 列表组织）
  3. 关键金句或方法论要点
- **每个核心概念配 1 个案例**：
  - 优先使用原书已给的案例
  - 必要时补充生活化类比（看病 / 做饭 / 报销 / 旅行 等）
- **保留关键术语英文**：避免翻译损失（如 `ReAct`、`Code Agent`、`SubAgent`）
- **不做评价、不做推断性结论**：仅复述与重组原文观点

### 4. 文档输出规范

#### 文件命名

```
{章节标识}-{章节标题}.md
```

示例：
- `第1章-从提示词到上下文工程.md`
- `chapter-3-deep-research.md`

#### Markdown 结构模板

```markdown
# 第N章《标题》内容总结与案例说明

## 一、章节主线
[演进图 / 一句话总览]

## 二、核心概念
### 概念A
- 定义
- 案例

### 概念B
...

## 三、案例与对比
[对照表、流程示意]

## 四、知识图谱
[章节结构树]

## 五、关键金句
> 原书核心观点摘录
```

#### 排版规范

- 标题层级：`#` 章 → `##` 节 → `###` 小节 → `####` 子项
- 表格优先用于：多对象对比、流程步骤、属性清单
- 代码块用于：JSON 示例、配置片段、关键命令
- 不写注释、不写自述、不写"以上即为总结"这类废话

### 5. 输出确认

完成后向用户报告：

- 输出文件路径（Code Reference 链接）
- 行数 / 大小
- 内容结构概览

## 反例（不要做的事）

- 不要把整本 PDF 一次性加载到上下文再总结（容易 OOM 且不精准）
- 不要输出"以上是我对第 N 章的总结"这类客套话
- 不要省略案例除非用户明确说"不要案例"
- 不要修改原文观点或添加未经验证的信息
- 不要创建多余的文件（README、索引等）

## 依赖

- Python 3.x
- `pypdf`（PDF 读取，按需安装）
- `python-docx`（Word 读取，按需安装）

## 示例对话

**用户**：总结 `/path/to/book.pdf` 的第 2 章，并写入 `assets/abstract/`

**本 Skill 自动执行**：
1. 加载 PDF，定位第 2 章
2. 提取文本块
3. 按模板生成总结（带案例）
4. 写入 `assets/abstract/第2章-xxx.md`
5. 返回链接

**用户**：这个 PDF 第 5 章讲了什么？给我列个大纲就行

**本 Skill 自动执行**：
1. 加载 PDF，定位第 5 章
2. 提取目录与要点（不展开案例）
3. 直接在对话中输出大纲（不写入文件，因为用户没要求）

# Doc Summarizer（文档摘要与输出）

## Purpose

把**原始文档**（PDF 书籍章节 / 长文章 / md / txt / docx）一次性提炼为**结构化 Markdown 摘要**，输出含主线、核心概念、案例对照、知识图谱、关键金句的速查文档。

适用场景：技术学习者、知识工作者、读书会、AI 课程内容准备。

## Trigger

用户说以下任意一句时立即触发：

- "总结 / 摘要 / 提炼 / abstract 某个文档"
- "输出为文档 / 写入到文档 / 生成总结文档"
- 指定具体章节："第一章"、"第 3 章"、"chapter 2"
- "附带案例 / 示例 / 类比"以方便理解
- 提供 PDF/MD/TXT 文件 + "总结 / 提炼"

## Workflow

### 1. 解析输入

从用户消息中提取以下要素：

| 要素 | 默认行为 | 可覆盖 |
|------|---------|--------|
| 文档路径 | 用户必须提供 | 无 |
| 范围 | 整篇文档 | 用户指定章节、页码、关键词 |
| 输出目录 | `assets/abstract/` | 用户指定 |
| 输出语言 | 与用户输入语言一致 | 用户指定 |
| 是否配案例 | 是（默认开启） | 用户要求关闭 |

如关键信息缺失，**优先做合理推断并继续**，在最终输出中标注所做的假设。

### 2. 文档读取

按扩展名选择读取方式：

| 格式 | 方法 |
|------|------|
| `.pdf` | `pypdf` 库（如未安装：`pip install --user --break-system-packages pypdf`） |
| `.md` / `.txt` | 直接 `Read` 工具 |
| `.docx` | `python-docx` 库 |
| 代码文件 | `Read` 工具 |

**关键原则**：先**识别目录 / 大纲**，定位用户指定的章节，**只提取对应文本块**（避免一次性把整本书塞入上下文）。

```python
import pypdf
reader = pypdf.PdfReader('file.pdf')
# 先扫前 10 页找目录
for i in range(min(10, len(reader.pages))):
    text = reader.pages[i].extract_text()
    if '第' in text and '章' in text:
        # 定位章节边界
        break
```

### 3. 内容提炼原则

- **三层结构**：
  1. 章节主线（一句话 + 演进图）
  2. 核心概念（用对照表 / 列表组织）
  3. 关键金句或方法论要点
- **每个核心概念配 1 个案例**：
  - 优先使用原书已给的案例
  - 必要时补充生活化类比（看病 / 做饭 / 报销 / 旅行 等）
- **保留关键术语英文**：避免翻译损失（如 `ReAct`、`Code Agent`、`SubAgent`）
- **不做评价、不做推断性结论**：仅复述与重组原文观点

### 4. 文档输出规范

#### 文件命名

```
{章节标识}-{章节标题}.md
```

示例：
- `第1章-从提示词到上下文工程.md`
- `chapter-3-deep-research.md`
- `第2章-上下文的元驱动.md`

#### Markdown 结构模板

```markdown
# 第N章《标题》内容总结与案例说明

## 一、章节主线
[演进图 / 一句话总览]

## 二、核心概念
### 概念A
- 定义
- 案例

### 概念B
...

## 三、案例与对比
[对照表、流程示意]

## 四、知识图谱
[章节结构树]

## 五、关键金句
> 原书核心观点摘录
```

#### 排版规范

- 标题层级：`#` 章 → `##` 节 → `###` 小节 → `####` 子项
- 表格优先用于：多对象对比、流程步骤、属性清单
- 代码块用于：JSON 示例、配置片段、关键命令
- 不写注释、不写自述、不写"以上即为总结"这类废话

### 5. 输出确认

完成后向用户报告：

- 输出文件路径（Code Reference 链接）
- 行数 / 大小
- 内容结构概览

## Rules

- **绝不**把整本 PDF 一次性加载到上下文再总结（容易 OOM 且不精准）
- **绝不**大段照抄原文——总结 + 重组 + 类比才是价值
- **绝不**省略案例除非用户明确说"不要案例"
- **绝不**修改原文观点或添加未经验证的信息
- **绝不**输出"以上是我对第 N 章的总结"这类客套话
- **绝不**创建多余的文件（README、索引等）
- **绝不**反复询问已能从上下文推断的信息（默认推进 + 标注假设）
- 用户没说英文就**默认中文**
- 每个核心概念**必须**配 1 个案例或生活类比
- 关键术语**保留英文原文**避免翻译损失

## Anti-patterns

- ❌ 不要把"PDF 提取文本"和"内容提炼"混为一步——分两步清晰
- ❌ 不要用 `cat` 读 PDF（Read 工具也读不了 binary），必须用 `pypdf`
- ❌ 不要把类比堆成段——一个概念配**一个**最贴的类比即可
- ❌ 不要在摘要里写"接下来我们看下一节"这种过渡——直接列要点
- ❌ 不要给"用户友好建议"或"操作指引"——只做内容提炼
- ❌ 不要给章节超过 3 层嵌套标题（信息密度下降）

## Quick Reference（速查）

| 输入类型 | 提取方式 | 备注 |
|---------|---------|------|
| PDF | `pypdf` | 按页提取，先扫目录定位章节 |
| MD / TXT | `Read` 工具 | 直接读全文 |
| DOCX | `python-docx` | 按需安装 |
| 代码文件 | `Read` 工具 | 整文件直接读 |

| 输出类型 | 字数参考 | 章节数 |
|---------|---------|--------|
| 章节摘要 md | 3000~6000 字 | 5~8 节主线 |
| 单概念总结 | 300~800 字 | 1 概念 + 1 案例 |
| 速查表 | 200~500 字 | 表格为主 |

## 依赖

- Python 3.x
- `pypdf`（PDF 读取，按需安装：`pip install --user --break-system-packages pypdf`）
- `python-docx`（Word 读取，按需安装）

## 示例对话

**用户**：总结 `/path/to/book.pdf` 的第 2 章，并写入 `assets/abstract/`

**本 Skill 自动执行**：
1. 用 pypdf 加载 PDF，扫描前 10 页定位第 2 章起止页
2. 提取该章节文本块
3. 按模板生成总结（带案例 + 知识图谱 + 关键金句）
4. 写入 `assets/abstract/第2章-xxx.md`
5. 返回 Code Reference 链接 + 行数 + 结构概览

**用户**：这个 PDF 第 5 章讲了什么？给我列个大纲就行

**本 Skill 自动执行**：
1. 加载 PDF，定位第 5 章
2. 提取目录与要点（不展开案例）
3. 直接在对话中输出大纲（不写入文件，因为用户没要求）
